## 첫 번째 댓글 (반드시 직접 작성)

```
Hey HN, I built this because I run a drug interaction checker (pillright.com) serving
Korean users, and I kept finding LLMs generating dangerously wrong medical content.

The problem: An LLM will confidently say "Take 50,000 IU of Vitamin D daily to cure
depression" — that's 12x the safe limit with an unsubstantiated cure claim. Generic
guardrail tools (NeMo, Guardrails AI) catch toxicity and PII, but they know nothing
about medicine.

llm-medical-guard is a Python library that validates medical content with 8 specialized
checks:

- Banned expressions ("miracle cure", "no side effects")
- Dosage validation (10 supplements with NIH-sourced safe limits)
- Drug interactions (15 FDA/NIH-documented dangerous combos)
- Disclaimer enforcement
- Source attribution (FDA, NIH, WHO, etc.)
- Tone analysis (educational vs. fear-mongering vs. promotional)
- Unsubstantiated claim detection
- Brand mention flagging

Key design decisions:

1. Rule-based, not LLM-based. 64µs per check, 15,000+ checks/sec. Zero API cost.
   Deterministic and auditable. Fast enough for real-time streaming validation.

2. One dependency (pyyaml). No heavy ML frameworks. pip install and go.

3. 5 languages (en/ko/ja/zh/es). Medical misinformation is a global problem.
   Most guardrail tools are English-only.

4. Developer-first: CLI tool, pytest plugin, LangChain/OpenAI integrations,
   streaming support, SVG verification badges.

Limitations I want to be upfront about:
- 15 drug interactions is a demo-level database, not comprehensive (DrugBank has tens
  of thousands). It's extensible via YAML/custom checks.
- Regex-based detection can be bypassed with creative phrasing. This is the fast first
  pass, not a replacement for expert review.
- This is a dev tool, not medical advice.

I'd love feedback on:
- What checks would you add for your use case?
- Anyone working in health-tech AI who would use this?
- Ideas for reducing false positives in the tone analysis?

Python 3.10+, MIT licensed. pip install llm-medical-guard.
```
